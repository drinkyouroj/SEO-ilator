## Data Flow

1. **Ingestion** вҖ” Articles enter via sitemap crawl, file upload, or API push
2. **Normalization** вҖ” All inputs are parsed and stored as `Article` records in PostgreSQL
3. **Analysis** вҖ” User triggers an analysis run; the registry fans out to all enabled strategies
4. **Recommendations** вҖ” Each strategy returns typed `Recommendation[]` with severity and suggestions
5. **Review** вҖ” Dashboard presents recommendations; user accepts or dismisses each one

## Embedding Pipeline

The crosslink strategy's semantic similarity mode uses an abstracted embedding provider
(`src/lib/embeddings/`). Currently supports OpenAI and Cohere. The provider is selected
via environment config (`EMBEDDING_PROVIDER`).

Embeddings are stored on the `Article` model and recomputed on content change.
